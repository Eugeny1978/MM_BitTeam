Информация по tradesOfUser
1. Пример запроса со всеми параметрами:
/trade/api/ccxt/tradesOfUser?order[timestamp]=DESC&order[id]=ASC&offset=0&limit=10&pairId=29&pair=btc_usdt&startTime=2024-02-26

2. Описание параметров:
  - limit — количество элементов. По умолчанию равен 10. Не ограничен 
  - order — массив колонок, по которым делает сортировка. Принимает 2 типа значений: DESC и ASC.
  - offset — навигация со смещением. По умолчанию равен 0
  - order — сортировка по колонкам. По умолчанию равен createdAt=DESC. Разрешенные колонки для сортировки — все, участвующие в выводе. Разрешенные значения: DESC и ASC. 
  - pairId — ИД валютной пары
  - startTime — новое поле. Фильтрует по полю createdAt, начиная с определенной даты или времени в формате timestampz. Пример значения 2024-02-26T07:35:00.554Z'
  - endTime — новое поле. Фильтрует по полю createdAt, заканчивая к определенной дате или времени в формате timestampz. Пример значения `2024-02-26T07:35:00.554Z'

В общем, обновил выше информацию по этим двум эндпойнтам. Список изменений:
- добавил новые параметры startTime и endTime для /tradesOfUser
- поправил баг c параметром limit на /tradesOfUser, который отдельно делал неправильный запрос на подсчет элементов выдачи
- поправил баг с параметром order для /ordersOfUser, который ломал запрос при добавлении несколько колонок для фильтрации
- добавил новые 3 валидные колонки(side, price, type) для where для /ordersOfUser
